🧠 Capacidades de modelos de razonamiento

Texto, análisis, código, visión, funciones y más. Ejemplos listos para copiar.

Análisis de texto

Resúmenes, clasificaciones, extracción de datos, redacción y traducción con control de tono y formato.

Actúa como analista. Resume el siguiente texto en 5 viñetas y agrega riesgos y oportunidades. Formato: JSON con keys [bullets, riesgos, oportunidades]. Texto: …
Código & Depuración

Generación, explicación y refactor de código; casos de prueba y migraciones.

Actúa como revisor senior. Explica el siguiente error y propone fix minimal. Entrega: diff en formato unified + 3 pruebas unitarias. Código: …
Razonamiento estructurado

Divide problemas en pasos, planea y verifica. Pide resúmenes de razonamiento de alto nivel.

Resuelve este problema en pasos verificables. Da un plan breve y luego la respuesta final con comprobaciones.
Visión (texto+imagen)

Describe, compara, extrae texto, sugiere mejoras de diseño y QA visual.

Analiza esta captura: detecta errores de UX, ordena por severidad y propone 3 quick wins. Entrega en tabla Markdown.
Funciones / Herramientas

Conexión a APIs (function calling), bases de datos, RAG y acciones automatizadas.

Cuando necesites datos de ventas, llama a getSales({region, from, to}). Si faltan parámetros, pregúntame primero. Responde con KPIs + insights.
RAG (citas & fuentes)

Recupera documentos y cita fuentes. Útil para investigación, compliance y soporte.

Usa RAG sobre mi carpeta "Políticas 2025". Devuelve respuesta + 3 citas relevantes con enlace. Señala incertidumbres.
Datos & Tablas

Construye tablas, valida datos, genera CSV/JSON, crea gráficos simples.

Con este CSV, crea una tabla de 5 KPIs con definición, fórmula y ejemplo. Entrega también un dataset limpio (CSV).
Idiomas & Tono

Control sobre formalidad, jerga y estilos (marketing, académico, técnico, casual).

Reescribe este texto a tono profesional y conciso (máx. 120 palabras) y agrega una versión casual para redes.

✍️ Prompting: frameworks y plantillas

Arma pedidos claros y repetibles. Itera y mide.

Framework ROCFT

Rol • Objetivo • Contexto • Formato • Tono.

Tip: Añade "Evita contenido sensible" y "Si falta info, pregunta".
Actúa como [rol]. Objetivo: [objetivo]. Contexto: [contexto]. Formato: [formato]. Tono: [tono]. Restricciones: [límites]. Si falta info, pregunta.
Plantillas útiles
  • Corrección/reescritura con criterios explícitos
  • Evaluador con rúbrica
  • Generador de datos sintéticos
  • Agente de QA para workflows

📏 Calidad

Haz: define inputs, salidas y criterios de éxito medibles.
Evita: pedir “magia”; sé concreto con formato, tono y límites.
Checklist
  • Fuentes/citas cuando aplica
  • Verificación rápida (self-check)
  • Incertidumbres explícitas

🛡️ Ética & Seguridad

Privacidad

Evita datos sensibles; usa datos ficticios cuando puedas.

Limitaciones

Señala incertidumbres y evita afirmaciones categóricas sin respaldo.
